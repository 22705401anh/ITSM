  1. {% extends "base.html" %}
  2. {% extends "base.html" %}

Root Cause:
  The file had a blank line at the beginning (line 1)
  When regenerating the template, the {% extends %} tag wasn't placed correctly
  This created confusion in the Jinja2 template engine

?? HOW TO AVOID THIS IN THE FUTURE

When working with Jinja2 templates:
  ? Always start with `{% extends %}` on line 1 (no blank lines before)
  ? Keep blocks properly nested
  ? Ensure only ONE extends statement per template
  ? If error occurs, regenerate the entire template file
  ? Don't mix extends from different calls
